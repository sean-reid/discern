# Discern

A swipe game. You see a photo, you decide: real or fake?

Swipe right if you think it's a real photo. Swipe left if you think it was generated by AI. Your rating goes up when you're right, down when you're wrong. The better you get, the harder the photos.

## Stack

- **Frontend**: Next.js 15, Framer Motion, Tailwind CSS 4, Zustand
- **Backend**: Cloudflare Workers (via OpenNext), D1 (SQLite), R2 (image storage)
- **Image pipeline**: Automated sourcing from Unsplash/Pexels/Flickr + AI generation via Cloudflare Workers AI and Pollinations.ai
- **Difficulty**: Elo rating system that adapts per-user and per-image

## Project Structure

```
src/
  app/              # Next.js pages and API routes
    api/            # Game API (images, swipe, stats, health)
    play/           # Main game screen
    stats/          # Player stats
  components/       # SwipeCard, SwipeStack, ResultFlash, StatsBar, etc.
  lib/              # Elo engine, anti-cheat, image selection, types
  stores/           # Zustand game state
  hooks/            # Image preloader, device ID
pipeline/           # Cloudflare Worker for automated image ingestion
  src/sources/      # Unsplash, Pexels, Flickr API fetchers
  src/processing/   # Image validation, hashing, EXIF analysis
scripts/            # Seeding and maintenance scripts
migrations/         # D1 database schema
tests/              # Unit, integration, and E2E tests
```

## How It Works

1. The pipeline worker runs daily on cron, pulling real photos from stock APIs and generating AI images
2. Each image gets an Elo difficulty rating that adjusts based on how often players guess correctly
3. Players get matched with images near their skill level
4. The API never sends the answer to the client until after a guess is submitted
